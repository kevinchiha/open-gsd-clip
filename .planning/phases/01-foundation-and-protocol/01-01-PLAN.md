1. **src/shared/logger.ts**: Create pino logger instance that writes to stderr (fd 2). This is critical -- stdout is reserved for JSON-RPC protocol in the plugin worker.
   - Export a `logger` instance configured with `name: '@open-gsd/clip'`
   - Level from `process.env.LOG_LEVEL ?? 'info'`
   - In development (`NODE_ENV=development`), use pino-pretty transport targeting stderr
   - In production, use `pino.destination(2)` for stderr
   - Export a `createChildLogger(name: string)` helper for component-specific loggers

2. **src/shared/errors.ts**: Base error class for the project.
   - `GsdError extends Error` with `name` auto-set from constructor name
   - `GsdOperationalError extends GsdError` for expected operational failures (bridge errors, parse errors, timeouts)
   - Both carry optional `cause` for error chaining

3. **src/shared/types.ts**: Shared utility types.
   - `Result<T, E = Error>` type: `{ ok: true; value: T } | { ok: false; error: E }`
   - `JsonValue`, `JsonObject` utility types for JSON-RPC message typing

4. **src/shared/logger.test.ts**: Verify logger writes to stderr, not stdout.
