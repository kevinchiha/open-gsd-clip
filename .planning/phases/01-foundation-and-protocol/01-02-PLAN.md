  <behavior>
    - Transport: onMessage callback receives parsed JSON objects from newline-delimited input
    - Transport: responses are written as newline-delimited JSON to output
    - Transport: malformed JSON lines produce JSON-RPC parse error response (code -32700)

    - Handler: "initialize" method returns manifest with id "@open-gsd/clip", apiVersion 1, version, displayName, capabilities
    - Handler: "health" method returns { status: "ok" }

    - Handler: "shutdown" method logs and returns { status: "shutting_down" }
    - Handler: unknown method returns JSON-RPC error code -32601 (Method not found)
    - Handler: request missing "jsonrpc": "2.0" returns JSON-RPC error code -32600 (Invalid Request)
    - Handler: notification requests (no id field) do not produce a response
  </behavior>
  <action>
1. **src/plugin/types.ts**: Define all JSON-RPC 2.0 types using Zod schemas:
   - `JsonRpcRequestSchema` (jsonrpc: "2.0", method: string, params?: unknown, id?: string|number|null)
   - `JsonRpcResponse`, `JsonRpcError` types
   - `RPC_ERRORS` constant object with standard error codes (-32700, -32600, -32601, -32602, -32603)
   - `PaperclipPluginManifestV1` interface with: id, apiVersion, version, displayName, description, categories, capabilities, entrypoints
   - Use `import { z } from 'zod'` (Zod 3 API, per research recommendation for @paperclipai/shared compatibility)

2. **src/plugin/rpc-transport.ts**: Line-delimited JSON framing layer.
   - `createTransport(input: Readable, output: Writable)` factory
   - Uses `readline.createInterface` with `crlfDelay: Infinity`
   - `onMessage(handler)`: for each line, parse JSON, call handler, write response + newline
   - On JSON parse error: write JSON-RPC parse error response
   - Skip empty lines
   - `close()` method to clean up readline
   - Use the shared logger (child logger named 'rpc-transport') for any logging

3. **src/plugin/rpc-transport.test.ts**: Unit tests using PassThrough streams.
   - Test: valid JSON line -> handler called with parsed object -> response written
   - Test: malformed JSON -> parse error response written
   - Test: empty line -> no handler call, no response
   - Test: multiple lines -> each processed independently

4. **src/plugin/rpc-handler.ts**: JSON-RPC method dispatcher.
   - `createRpcHandler()` returns an async function `(request: unknown) => Promise<unknown>`
   - Validates request against JsonRpcRequestSchema
   - Invalid request schema -> error -32600
   - Routes by method name to handler functions
   - Known methods: initialize, health, onEvent, getState, executeAction, registerTools, shutdown, configure
   - Unknown method -> error -32601
   - Notification (no id) -> handler runs but returns undefined (no response)
   - `initialize` handler returns `{ jsonrpc: "2.0", result: manifest, id }`
   - `health` handler returns `{ jsonrpc: "2.0", result: { status: "ok" }, id }`
   - `onEvent` handler logs event type, returns `{ jsonrpc: "2.0", result: { received: true }, id }`
   - Remaining methods return `{ jsonrpc: "2.0", result: { status: "not_implemented" }, id }`

5. **src/plugin/rpc-handler.test.ts**: Unit tests for all method handlers.
   - Test initialize returns manifest with correct fields
   - Test health returns ok
   - Test onEvent returns received: true
   - Test unknown method returns -32601
   - Test invalid request (missing jsonrpc) returns -32600
   - Test notification (no id) returns undefined
