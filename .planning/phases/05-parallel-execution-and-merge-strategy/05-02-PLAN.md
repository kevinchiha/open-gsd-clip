<objective>
Refactor PipelineRunner from sequential to parallel execution with worktree integration and ordered merging.

Purpose: This plan transforms the Phase 4 sequential pipeline into a parallel executor. When the execution plan contains groups with multiple phases, all phases in a group start simultaneously, each in its own git worktree. Completed results merge in roadmap order via MergeQueue. The refactored runner subsumes sequential behavior -- groups of size 1 behave identically to Phase 4.

Output: Modified PipelineRunner with parallel dispatch, updated barrel export.
</objective>

       ```typescript
       private async startPhase(phaseNumber: number): Promise<void> {

         }
         await this.handlePhaseEvent(phaseNumber, { type: 'DEPENDENCIES_MET' });
       }
       ```

    5. **Add new private method findReadyPhases():**
       Follow the research code example. Returns all pending phases whose dependsOn are all in 'done' status. Uses this.phaseInputs for dependency data. Returns sorted array of phase numbers.
