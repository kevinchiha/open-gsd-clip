**Pipeline lifecycle:**
- `async start(projectPath: string, brief: string): Promise<void>`:
  1. Create initial pipeline state via `createInitialPipelineState(projectPath, brief)`
  2. Transition: `pipelineTransition(state, { type: 'START_PIPELINE' })`
  3. Validate transition
  4. Ensure agents exist via `ensureAgentsExist(services, projectPath, companyId)`
  5. Spawn CEO: `spawnAgent(services, companyId, agents.ceo.agentId, { role: 'ceo', projectPath, gsdCommand: '/gsd:new-project --auto', brief })`
  6. Track agent in health monitor
  7. Persist state

- `async handleAgentCompletion(run: { status: string; agentId: string; runId: string; issueId?: string }): Promise<void>`:
  1. Enqueue in SerialEventQueue
  2. Untrack from health monitor
  3. Find which phase has this issueId as activeAgentIssueId
  4. Clear agent on that phase
  5. If status=succeeded AND issueId: fetch comments, find last GSD_SIGNAL, map to PhaseEvent
  6. If status=failed: create STEP_FAILED event with 'fatal' type
  7. If no signal found on success: create STEP_FAILED with 'fatal' and 'no signal' message
  8. Special case: PROJECT_READY signal -> call handleProjectReady instead of handlePhaseEvent
  9. Dispatch to handlePhaseEvent(phaseNumber, phaseEvent)

- `private async handleProjectReady(): Promise<void>`:
  1. Transition pipeline: PROJECT_READY
  2. Read roadmap phases via gsd-tools bridge (or derive from config)
  3. Build execution plan
  4. Create PhaseState for each phase
  5. Transition pipeline: ANALYSIS_COMPLETE with execution plan
  6. Start first phase in execution order (send DEPENDENCIES_MET)
  7. Persist state

- `private async handlePhaseEvent(phaseNumber: number, event: PhaseEvent): Promise<void>`:
  1. Get phase state from pipeline
  2. Transition: `phaseTransition(phaseState, event)`
  3. Update phase in pipeline state
  4. Persist state
  5. Call `advancePhase(phaseNumber, newPhaseState)`

- `private async advancePhase(phaseNumber: number, phase: PhaseState): Promise<void>`:
  Switch on phase.status:
  - `discussing`: spawn discusser with `/gsd:discuss-phase ${phaseNumber} --auto`
  - `reviewing`: spawn CEO review (check revision count first -- if over limit, fail phase). Build review issue with `buildReviewIssueDescription` pointing to `.planning/phases/XX-*/XX-CONTEXT.md`.
  - `planning`: spawn planner with `/gsd:plan-phase ${phaseNumber}`
  - `executing`: spawn executor with `/gsd:execute-phase ${phaseNumber}`
  - `verifying`: spawn verifier with `/gsd:verify-work ${phaseNumber}`
  - `done`: call `onPhaseComplete(phaseNumber)`
  - `failed`: call `onPhaseFailed(phaseNumber, phase)`

- `private async onPhaseComplete(phaseNumber: number): Promise<void>`:
  1. Transition pipeline: PHASE_COMPLETED
  2. Check if all phases are done
  3. If all done: transition pipeline ALL_PHASES_DONE
  4. If not: find next pending phase in execution order, send DEPENDENCIES_MET
  5. Persist state

- `private async onPhaseFailed(phaseNumber: number, phase: PhaseState): Promise<void>`:
  1. Classify error via classifyError
  2. If retryable and retryCount < maxRetries:
     - Record retry decision in audit log
     - Send RETRY_PHASE to reset phase
     - Phase will re-enter advancePhase at pending -> discussing
  3. If not retryable or exhausted retries:
     - Record failure decision in audit log
     - Cascade failure to dependent phases
     - Transition pipeline: UNRECOVERABLE_ERROR if no remaining phases can proceed
