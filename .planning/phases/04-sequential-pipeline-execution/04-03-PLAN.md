  <name>Task 1: Implement serial event queue</name>
  <files>src/orchestrator/event-queue.ts, src/orchestrator/event-queue.test.ts</files>
  <behavior>
    - enqueue(handler) adds async handler to queue and returns promise that resolves when handler completes
    - Handlers execute one at a time, in order (serial)
    - If handler throws, the enqueue promise rejects but queue continues processing remaining items
    - Multiple concurrent enqueue calls are serialized (no parallel execution)
    - Queue drains automatically as items are added
  </behavior>
  <action>
Create `src/orchestrator/event-queue.ts`:

Implement `SerialEventQueue` class:
- `private processing = false`
- `private queue: Array<{ handler: () => Promise<void>; resolve: () => void; reject: (err: unknown) => void }> = []`
- `async enqueue(handler: () => Promise<void>): Promise<void>` -- wraps handler in a Promise, pushes to queue, calls `drain()`, returns the promise
- `private async drain(): Promise<void>` -- if already processing, return. Set processing=true. While queue has items: shift item, try/catch handler, resolve/reject. Set processing=false.

Create `src/orchestrator/event-queue.test.ts`:

Test groups:
1. **serial execution** -- enqueue 3 handlers that push to an array with a delay. Verify execution order matches enqueue order.
2. **error isolation** -- enqueue handler that throws, then handler that succeeds. Verify first rejects, second resolves.
3. **concurrent enqueue** -- fire multiple enqueue calls without await between them. Verify all complete in order.
4. **empty queue** -- enqueue one handler, await it, verify queue is idle.
  </action>
