  <name>Task 2: Implement error classification and retry with exponential backoff</name>
  <files>src/orchestrator/error-handler.ts, src/orchestrator/error-handler.test.ts</files>
  <behavior>
    - classifyError("context overflow exceeded") returns { type: 'context_overflow', retryable: false, maxRetries: 0 }
    - classifyError("ECONNRESET on socket") returns { type: 'transient', retryable: true, maxRetries: 3 }
    - classifyError("test assertion failed") returns { type: 'test_failure', retryable: true, maxRetries: 2 }
    - classifyError("CONFLICT in merge") returns { type: 'merge_conflict', retryable: true, maxRetries: 1 }
    - classifyError("unknown catastrophic error") returns { type: 'fatal', retryable: false, maxRetries: 0 }
    - calculateBackoffDelay(0, config) returns value in [0, baseDelayMs)
    - calculateBackoffDelay(1, config) returns value in [0, baseDelayMs*2)
    - calculateBackoffDelay(10, config) returns value in [0, maxDelayMs) -- capped
    - retryWithBackoff(fn, config) calls fn up to maxRetries+1 times on failure
    - retryWithBackoff(fn) returns immediately on first success
    - retryWithBackoff(fn) throws last error after all retries exhausted
  </behavior>
  <action>
Create `src/orchestrator/error-handler.ts`:

Import `ClassifiedError`, `RetryConfig`, `DEFAULT_RETRY_CONFIG` from `./types.js`.
Import `type ErrorType` from `../pipeline/types.js`.

Define ERROR_PATTERNS array with entries: { pattern: RegExp, type: ErrorType, retryable: boolean, maxRetries: number }:
- context_overflow: `/context.*(overflow|limit|too long|token)/i` -- not retryable
- test_failure: `/(test|assertion|expect).*(fail|error)/i` -- retryable, 2 retries
- merge_conflict: `/(merge conflict|CONFLICT|<<<<<<)/i` -- retryable, 1 retry
- transient: `/(ECONNRESET|ETIMEDOUT|rate.?limit|503|502|429)/i` -- retryable, 3 retries

`classifyError(message: string): ClassifiedError` -- iterate patterns, first match wins, default to 'fatal' (not retryable).

`calculateBackoffDelay(attempt: number, config: RetryConfig): number` -- formula: `Math.floor(Math.random() * Math.min(config.baseDelayMs * Math.pow(2, attempt), config.maxDelayMs))`. Full jitter variant.

`retryWithBackoff<T>(fn: () => Promise<T>, config?: RetryConfig): Promise<T>` -- loop from 0 to maxRetries, call fn, on error wait calculateBackoffDelay, rethrow last error. Use `setTimeout` from `node:timers/promises` for non-blocking delay.

Create `src/orchestrator/error-handler.test.ts`:

Test groups:
1. **classifyError** -- test each of the 5 error types with representative strings, including the default-to-fatal case
2. **calculateBackoffDelay** -- verify delay is in expected range [0, min(base*2^n, max)), verify cap at maxDelayMs, verify attempt 0 gives [0, baseDelayMs)
3. **retryWithBackoff** -- test success on first try, success after transient failures, throws after exhausting retries, verify fn called correct number of times

For backoff delay tests: seed Math.random or test statistical bounds (run 100 times, verify all in range).
For retryWithBackoff tests: use `vi.fn()` to count calls, mock setTimeout via `vi.useFakeTimers()`.
