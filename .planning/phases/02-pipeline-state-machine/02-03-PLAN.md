Define Zod schemas per RESEARCH.md Pattern 4:

```typescript
import { z } from 'zod';

const StepTimingSchema = z.object({
  startedAt: z.string().datetime().nullable(),
  completedAt: z.string().datetime().nullable(),
});

const PhaseErrorSchema = z.object({
  type: z.enum(['transient', 'context_overflow', 'test_failure', 'merge_conflict', 'fatal']),
  message: z.string(),
  retryCount: z.number().int().min(0),
  lastAttemptAt: z.string().datetime().nullable(),
});

const FailureCascadeInfoSchema = z.object({
  rootCausePhase: z.number().int().positive(),
  failureType: z.string(),
  errorSummary: z.string(),
});

const PhaseStateSchema = z.object({
  phaseNumber: z.number().int().positive(),
  status: z.enum(['pending', 'discussing', 'reviewing', 'planning', 'executing', 'verifying', 'done', 'failed']),
  stepTimings: z.record(z.string(), StepTimingSchema),
  activeAgentIssueId: z.string().nullable(),
  error: PhaseErrorSchema.nullable(),
  failureCascade: FailureCascadeInfoSchema.nullable(),
});

const ExecutionPlanSchema = z.object({
  groups: z.array(z.array(z.number().int().positive())),
  phaseOrder: z.array(z.number().int().positive()),
});

const PipelineStateSchema = z.object({
  status: z.enum(['idle', 'initializing', 'analyzing', 'running', 'paused', 'completed', 'failed']),
  phases: z.array(PhaseStateSchema),
  executionPlan: ExecutionPlanSchema.nullable(),
  startedAt: z.string().datetime().nullable(),
  completedAt: z.string().datetime().nullable(),
  lastTransitionAt: z.string().datetime(),
  projectPath: z.string(),
  brief: z.string(),
});
```

Adjust schemas if the actual types.ts has different field names or shapes. The schemas must match exactly.

**Functions:**
- `serialize(state: PipelineState): string` -- `JSON.stringify(state)`. Simple but explicit function for consistency.
- `deserialize(json: string): Result<PipelineState, z.ZodError>` -- `JSON.parse(json)` then `PipelineStateSchema.safeParse()`. Returns `Result<PipelineState, z.ZodError>`.

Export all schemas (consumers may need them for partial validation) and both functions.

**Important Zod 3.25 note:** Use `z.string().datetime()` for ISO datetime validation. If Zod 3.25 uses different syntax (e.g., `z.iso.datetime()` or `z.string().pipe(z.coerce.date())`), check the installed version's API. The key requirement is that ISO 8601 datetime strings round-trip correctly.
