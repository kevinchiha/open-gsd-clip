Create `src/pipeline/types.ts` with all state types per RESEARCH.md:

**Pipeline types:**
- `PipelineStatus`: 'idle' | 'initializing' | 'analyzing' | 'running' | 'paused' | 'completed' | 'failed'
- `PipelineEvent`: discriminated union with types START_PIPELINE, PROJECT_READY, ANALYSIS_COMPLETE, PHASE_COMPLETED, PHASE_FAILED, ALL_PHASES_DONE, PAUSE_REQUESTED, RESUME_REQUESTED, UNRECOVERABLE_ERROR
- `PipelineState`: status, phases (PhaseState[]), executionPlan (ExecutionPlan | null), startedAt/completedAt/lastTransitionAt (string | null), projectPath, brief
- `TransitionResult<S>`: generic { state: S; valid: boolean; description?: string }

**Phase types:**
- `PhaseStatus`: 'pending' | 'discussing' | 'reviewing' | 'planning' | 'executing' | 'verifying' | 'done' | 'failed'
- `PhaseEvent`: discriminated union with types DEPENDENCIES_MET, STEP_COMPLETED, STEP_FAILED, APPROVED, REVISION_NEEDED, RETRY_PHASE, AUTO_FAIL
- `PhaseState`: phaseNumber, status, stepTimings (Record<string, StepTiming>), activeAgentIssueId (string | null), error (PhaseError | null), failureCascade (FailureCascadeInfo | null)
- `ErrorType`: 'transient' | 'context_overflow' | 'test_failure' | 'merge_conflict' | 'fatal'
- `PhaseError`: { type: ErrorType; message: string; retryCount: number; lastAttemptAt: string | null }
- `FailureCascadeInfo`: { rootCausePhase: number; failureType: string; errorSummary: string }
- `StepTiming`: { startedAt: string | null; completedAt: string | null }

**Resolver input types (used by Plan 02):**
- `PhaseInput`: { phaseNumber: number; dependsOn: number[] }
- `ExecutionPlan`: { groups: number[][]; phaseOrder: number[] }

Create `src/pipeline/fsm.ts`:
- `pipelineTransition(state: PipelineState, event: PipelineEvent): TransitionResult<PipelineState>` -- pure function using switch on state.status, then switch on event.type. Returns { valid: false } for invalid transitions. Updates lastTransitionAt on every valid transition (use ISO string).
- `createInitialPipelineState(projectPath: string, brief: string): PipelineState` -- returns idle state with empty phases array, null timestamps, lastTransitionAt set to now.
- `cascadeFailure(state: PipelineState, failedPhase: number, error: PhaseError, dependents: Map<number, number[]>): PipelineState` -- BFS cascade with visited set per RESEARCH.md Pattern. Skips already-failed and done phases. Sets FailureCascadeInfo on each auto-failed phase.

Use `.js` extensions on all relative imports. Do NOT use Map in serializable state (use it only as function parameter in cascadeFailure where it's transient). Do NOT mutate state -- always spread to create new objects.

    - verifying + STEP_FAILED transitions back to executing (retry loop -- resets executing timing)
    - Any active step + STEP_FAILED transitions to failed (with error info)
    - failed + RETRY_PHASE transitions to pending (clears error, resets timings)
    - done rejects all events
    - failed rejects events other than RETRY_PHASE
    - pending + AUTO_FAIL transitions to failed (with cascade info)
    - activeAgentIssueId is set/cleared via SET_AGENT and CLEAR_AGENT events
    - createInitialPhaseState(phaseNumber) returns pending state with empty timings

- `phaseTransition(state: PhaseState, event: PhaseEvent): TransitionResult<PhaseState>` -- looks up transition in table, applies timing updates, handles error context on STEP_FAILED, clears error/cascade on RETRY_PHASE.
- `createInitialPhaseState(phaseNumber: number): PhaseState` -- returns pending state with empty stepTimings record, null error/cascade/agentId.
- Export `PHASE_TRANSITIONS` for inspection/testing.

**Timing logic (per user decisions):**
- When entering a step: set that step's `startedAt` to ISO now, clear `completedAt`
- When leaving a step: set that step's `completedAt` to ISO now
- On backward transition (e.g., verifying -> executing for retry, reviewing -> discussing for revision): reset target step's `startedAt` to now, clear `completedAt`. Previous step's timing remains as historical data.
- On RETRY_PHASE (failed -> pending): reset ALL step timings to null

**Agent tracking:**
- Add SET_AGENT and CLEAR_AGENT event types to PhaseEvent in types.ts. SET_AGENT carries agentIssueId string. These events do NOT change the phase status -- they only update activeAgentIssueId. The transition function handles them as metadata-only updates (return { valid: true } with same status but updated agentId).
