  <behavior>
    - runner.pause() transitions pipeline state to 'paused' and calls notificationService.notify('pipeline_paused')
    - runner.pause() throws if pipeline not started
    - runner.pause() throws if pipeline not in 'running' state
    - runner.resume() transitions pipeline state to 'running' and calls notificationService.notify('pipeline_resumed')
    - runner.resume() restarts pending phases whose dependencies are met
    - runner.retryPhase(N) throws if phase N is not in 'failed' state
    - runner.retryPhase(N) resets phase to pending and triggers DEPENDENCIES_MET
    - runner.resolveEscalation(id, decision) resolves pending escalation, records audit, and advances phase
    - runner.resolveEscalation throws for unknown escalation ID
    - When DECISION_NEEDED signal arrives, an EscalationRecord is created, notificationService.notify('escalation') is called, and the phase does NOT advance further (waits for user)
    - advancePhase does NOT spawn new agents when pipeline status is 'paused'
    - TokenTracker.recordUsage is called from heartbeat events with token data
    - notificationService.notify is called on phase_completed, phase_failed, pipeline_completed, pipeline_failed transitions
  </behavior>
  <action>
    1. Modify `src/orchestrator/pipeline-runner.ts`:

       a. Add new imports at top:
          - `import { NotificationService } from '../notifications/notification-service.js'`
          - `import { TokenTracker } from './token-tracker.js'`
          - `import type { EscalationRecord } from './types.js'`
          - `import { randomUUID } from 'node:crypto'`

       b. Add new private fields to PipelineRunner class:
          - `private notificationService: NotificationService | null = null`
          - `private readonly tokenTracker = new TokenTracker()`
          - `private readonly pendingEscalations = new Map<string, EscalationRecord>()`

       c. Add `setNotificationService(service: NotificationService): void` method (called after construction, before start).

       d. Add `pause()` method:
          - Throw if `!this.state`
          - Call `pipelineTransition(this.state, { type: 'PAUSE_REQUESTED' })`
          - If invalid, throw with description
          - Update state, persist, call `notificationService?.notify({ type: 'pipeline_paused' })`

       e. Add `resume()` method:
          - Throw if `!this.state`
          - Call `pipelineTransition(this.state, { type: 'RESUME_REQUESTED' })`
          - If invalid, throw
          - Update state, then iterate pending phases and start those with dependencies met
          - Persist, call `notificationService?.notify({ type: 'pipeline_resumed' })`

       f. Add `retryPhase(phaseNumber: number, _fromStep?: string)` method:
          - Throw if pipeline not started or phase not found or phase not failed
          - Dispatch RETRY_PHASE then DEPENDENCIES_MET via handlePhaseEvent

       g. Add `resolveEscalation(escalationId: string, decision: string)` method:
          - Find escalation in pendingEscalations map, throw if not found
          - Set resolvedAt and resolution on escalation
          - Remove from map
          - Record audit entry with decisionType: 'quality_gate'
          - Call advancePhaseAfterDecision to resume the phase

       h. Add private `handleDecisionNeeded(phaseNumber: number, signal: DecisionNeededSignal)` method:
          - Create EscalationRecord with randomUUID() id
          - Store in pendingEscalations map
          - Call notificationService?.notify({ type: 'escalation', phaseNumber, context: signal.context, options: signal.options })
          - Record audit log entry
          - Do NOT advance phase (phase waits for user resolution)

       i. Add private `advancePhaseAfterDecision(phaseNumber: number, decision: string)` method:
          - Construct STEP_COMPLETED event (the CEO's decision is received; advance from reviewing to planning)
          - Dispatch via handlePhaseEvent

       j. Modify existing `advancePhase()` method:
          - At the TOP of the method (before the switch), add: `if (this.state?.status === 'paused') { log.info({ phaseNumber }, 'Pipeline paused, deferring agent spawn'); return; }`
          - This prevents new agents from being spawned while paused (Pitfall 5 from research)

       k. Modify `processAgentCompletion()`:
          - After parsing signal, before handling: if signal.type === 'DECISION_NEEDED', call handleDecisionNeeded and return
          - In the heartbeat.run.status handling section (recordActivity path), extract token data if present and call tokenTracker.recordUsage

       l. Add notification hooks in existing methods:
          - In `onPhaseComplete()`: call `notificationService?.notify({ type: 'phase_completed', phaseNumber })`
          - In `onPhaseFailed()`: call `notificationService?.notify({ type: 'phase_failed', phaseNumber, error: errorMessage })`
          - In `onPhaseComplete()` when allSucceeded: call `notificationService?.notify({ type: 'pipeline_completed', totalPhases: this.state.phases.length })`
          - In `start()`: call `notificationService?.notify({ type: 'pipeline_started', projectPath })`

       m. Add `getTokenSummary()` public method that delegates to tokenTracker.getSummary().
       n. Add `getPendingEscalations()` public method that returns Array.from(pendingEscalations.values()).
       o. Add private `areDependenciesMet(phaseNumber: number): boolean` method -- check if all phases that this phase depends on are 'done' (needed by resume()).
