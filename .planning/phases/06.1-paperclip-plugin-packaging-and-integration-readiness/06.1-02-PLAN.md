<task type="auto">
  <name>Task 1: Create tsup.config.ts and update package.json</name>
  <files>tsup.config.ts, package.json</files>
  <action>
1. Create `tsup.config.ts` at project root:
   ```typescript
   import { defineConfig } from 'tsup';

   export default defineConfig({
     entry: {
       'manifest': 'src/plugin/manifest.ts',
       'plugin/worker': 'src/plugin/worker.ts',
       'index': 'src/index.ts',
     },
     format: ['esm'],
     target: 'node22',
     platform: 'node',
     outDir: 'dist',
     clean: true,
     sourcemap: true,
     dts: true,
     splitting: true,
     external: [
       '@paperclipai/shared',
       'execa',
       'js-yaml',
       'pino',
       'zod',
     ],
   });
   ```

   Key decisions (from research):
   - `entry` uses map syntax to control output paths: `'plugin/worker'` produces `dist/plugin/worker.js` matching manifest's `entrypoints.worker` path
   - `format: ['esm']` only -- package is `"type": "module"`, no CJS needed
   - `external` lists all runtime dependencies to prevent bundling (Pitfall 4 from research)
   - `splitting: true` enables code splitting for shared chunks between entry points
   - `dts: true` for TypeScript consumers, `sourcemap: true` for debugging

2. Update `package.json` -- add the following fields (keep all existing fields unchanged):
   - Add `"main": "./dist/index.js"`
   - Add `"types": "./dist/index.d.ts"`
   - Add `"exports"` field:
     ```json
     "exports": {
       ".": {
         "types": "./dist/index.d.ts",
         "import": "./dist/index.js"
       }
     }
     ```
   - Add `"files": ["dist"]`
   - Add `"paperclipPlugin"` field:
     ```json
     "paperclipPlugin": {
       "manifest": "./dist/manifest.js",
       "worker": "./dist/plugin/worker.js"
     }
     ```
   - Add `"prepublishOnly": "npm run build"` to scripts (ensures dist/ in npm tarball)
   - Add `"prepare": "npm run build"` to scripts (for local `npm link` workflows)

   Do NOT change: name, version, type, description, license, engines, dependencies, devDependencies, existing scripts (build, dev, test, test:watch, test:coverage, lint, lint:fix, typecheck).
  </action>
  <verify>
    <automated>npm run build && test -f dist/manifest.js && test -f dist/plugin/worker.js && test -f dist/index.js && echo "Build structure OK"</automated>
  </verify>
